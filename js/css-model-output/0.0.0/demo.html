CSS Model: Output w/ Binding

One-way Binding

When the first input is changed, data is pushed to the custom property '--input-foo' via the 'data-custom-property' attribute.

When the custom propety '--input-foo' changes, data is pulled into the second input via the 'data-custom-property-output' attribute.

Data is also pulled into the span. By default, it would be set to the "innerText" attribut, but we set the "data-custom-property-write-key" to "DANGEROULSY_SET_innerHTML" to allow HTML content.

Note that changing the second input does not propagate to the other two elements.

Two-way Binding

When data in either input is updated, the change is propagated to the other element as both reference the the same variable, '--input-bar', via the 'data-custom-property-bind' atttribute.
